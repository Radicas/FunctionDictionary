# 产品开发路线图

## 1. 产品愿景

**核心目标**：打造一个智能化的代码理解和知识管理工具，帮助开发者快速理解开源项目，降低上手难度，积累代码知识资产。

**核心价值**：
- 🚀 **快速理解**：通过AI自动分析代码，生成易懂的说明文档
- 📊 **可视化呈现**：自动生成架构图、流程图、关系图
- 💾 **知识沉淀**：将理解的知识持久化存储，形成个人知识库
- 🔍 **智能检索**：快速查找和复用已有知识

### 4.1 当前数据模型

```sql

### 4.2 扩展数据模型（v2.0）

```sql
-- 项目表
CREATE TABLE projects (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    name TEXT NOT NULL,
    path TEXT NOT NULL UNIQUE,
    description TEXT,
    language TEXT,
    framework TEXT,
    status TEXT DEFAULT 'active',
    create_time DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
    update_time DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP
);

-- 文件表
CREATE TABLE files (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    project_id INTEGER NOT NULL,
    file_path TEXT NOT NULL,
    file_name TEXT NOT NULL,
    file_type TEXT,
    relative_path TEXT,
    line_count INTEGER,
    language TEXT,
    status TEXT DEFAULT 'active',
    create_time DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (project_id) REFERENCES projects(id),
    UNIQUE(project_id, file_path)
);

-- 函数表（扩展）
CREATE TABLE functions (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    project_id INTEGER,
    file_id INTEGER,
    function_name TEXT NOT NULL,
    full_name TEXT,
    signature TEXT,
    return_type TEXT,
    parameters TEXT,
    description TEXT,
    code_snippet TEXT,
    line_start INTEGER,
    line_end INTEGER,
    complexity INTEGER,
    tags TEXT,
    category TEXT,
    ai_analysis TEXT,
    custom_notes TEXT,
    status TEXT DEFAULT 'active',
    create_time DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
    update_time DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (project_id) REFERENCES projects(id),
    FOREIGN KEY (file_id) REFERENCES files(id)
);

-- 类/结构体表
CREATE TABLE classes (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    project_id INTEGER NOT NULL,
    file_id INTEGER NOT NULL,
    class_name TEXT NOT NULL,
    full_name TEXT,
    base_classes TEXT,
    description TEXT,
    line_start INTEGER,
    line_end INTEGER,
    ai_analysis TEXT,
    create_time DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (project_id) REFERENCES projects(id),
    FOREIGN KEY (file_id) REFERENCES files(id)
);

-- 模块表
CREATE TABLE modules (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    project_id INTEGER NOT NULL,
    module_name TEXT NOT NULL,
    module_path TEXT,
    description TEXT,
    ai_analysis TEXT,
    create_time DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (project_id) REFERENCES projects(id)
);

-- 依赖关系表
CREATE TABLE dependencies (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    source_type TEXT NOT NULL,
    source_id INTEGER NOT NULL,
    target_type TEXT NOT NULL,
    target_id INTEGER NOT NULL,
    dependency_type TEXT,
    description TEXT,
    create_time DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP
);

-- 图表数据表
CREATE TABLE diagrams (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    project_id INTEGER NOT NULL,
    diagram_type TEXT NOT NULL,
    diagram_name TEXT NOT NULL,
    mermaid_code TEXT NOT NULL,
    description TEXT,
    create_time DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (project_id) REFERENCES projects(id)
);

-- 标签表
CREATE TABLE tags (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    tag_name TEXT NOT NULL UNIQUE,
    tag_type TEXT,
    color TEXT,
    description TEXT
);

-- 函数标签关联表
CREATE TABLE function_tags (
    function_id INTEGER NOT NULL,
    tag_id INTEGER NOT NULL,
    PRIMARY KEY (function_id, tag_id),
    FOREIGN KEY (function_id) REFERENCES functions(id),
    FOREIGN KEY (tag_id) REFERENCES tags(id)
);

-- 分析任务表
CREATE TABLE analysis_tasks (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    project_id INTEGER,
    task_type TEXT NOT NULL,
    target_path TEXT,
    status TEXT DEFAULT 'pending',
    progress INTEGER DEFAULT 0,
    total_count INTEGER,
    completed_count INTEGER,
    error_message TEXT,
    create_time DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
    start_time DATETIME,
    end_time DATETIME,
    FOREIGN KEY (project_id) REFERENCES projects(id)
);

-- 索引优化
CREATE INDEX idx_functions_project ON functions(project_id);
CREATE INDEX idx_functions_file ON functions(file_id);
CREATE INDEX idx_files_project ON files(project_id);
CREATE INDEX idx_dependencies_source ON dependencies(source_type, source_id);
CREATE INDEX idx_dependencies_target ON dependencies(target_type, target_id);
```

## 7. 技术实现要点

### 7.1 代码解析技术选型

**方案一：使用现有解析库**
- C++：Clang LibTooling
- Python：ast模块 / tree-sitter
- JavaScript：Acorn / Esprima
- 通用：tree-sitter（推荐）

**方案二：基于正则表达式**
- 优点：实现简单，速度快
- 缺点：准确度较低，难以处理复杂语法

**推荐方案**：
- 初期：正则表达式 + AI辅助（快速实现）
- 后期：集成tree-sitter（提高准确性）

### 7.2 AI分析优化策略

**提示词工程**：
```cpp
// 结构化提示词模板

**批量处理优化**：
- 合并多个函数一次性分析
- 使用上下文信息提高准确性
- 缓存常见模式的分析结果

### 7.3 性能优化策略

**数据库优化**：
- 合理使用索引
- 批量插入优化
- 查询优化
- 连接池管理

**内存优化**：
- 大文件流式处理
- 结果缓存
- 及时释放资源

**并发优化**：
- 文件扫描并行化
- AI请求异步化
- 任务队列管理

### 7.4 可视化技术选型

**方案一：Mermaid + QWebEngineView**（当前方案）
- 优点：简单易用，支持多种图表
- 缺点：交互性有限

**方案二：集成专业图表库**
- D3.js：功能强大，高度可定制
- Cytoscape.js：专门用于关系图
- GoJS：交互性强，商业授权

**推荐方案**：
- 初期：继续使用Mermaid
- 后期：根据需求考虑集成D3.js或Cytoscape.js
